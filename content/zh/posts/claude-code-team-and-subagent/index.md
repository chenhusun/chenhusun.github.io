## 什么是 Claude Code

Claude Code 是 Anthropic 推出的官方命令行工具（CLI），它让开发者可以在终端中直接与 Claude 交互，完成代码编写、调试、重构、代码审查等软件工程任务。与传统的 AI 对话不同，Claude Code 拥有丰富的工具集：它可以读写文件、执行 shell 命令、搜索代码库、与 Git 和 GitHub 交互，甚至能自主规划和执行复杂的多步骤任务。

## Team 协作

### Team Plan 概述

Claude Code 提供了 Team 订阅计划，专为团队协作场景设计。Team Plan 不仅仅是简单的多人共享，它从根本上考虑了团队开发的协作需求：

- **共享上下文**：团队成员可以通过 `CLAUDE.md` 文件共享项目级别的上下文和约定，确保每个人使用 Claude Code 时都遵循相同的编码规范和项目约定
- **统一配置**：Team 级别的设置和权限管理，让管理员可以为整个团队配置统一的工具权限和行为规范
- **协作工作流**：支持多人在同一代码库上并行工作，Claude Code 能理解团队的 Git 工作流，协助处理分支管理、代码审查和 PR 创建

### CLAUDE.md — 团队的共享记忆

`CLAUDE.md` 是 Claude Code 的一个核心概念，它本质上是一个 Markdown 文件，放置在项目根目录中，用于告诉 Claude Code 关于项目的重要信息：

```markdown
# 项目约定

- 使用 TypeScript 严格模式
- 所有 API 端点需要添加错误处理
- commit message 遵循 Conventional Commits 规范
- 测试覆盖率不低于 80%
```

在团队场景下，`CLAUDE.md` 成为了团队共享知识的载体。新成员加入时，Claude Code 会自动读取这些约定，确保产出的代码与团队标准一致。

## Sub-Agent 机制

Sub-Agent（子代理）是 Claude Code 最强大的能力之一。它本质上是一个任务委派系统：当面对复杂任务时，主 Agent 可以启动多个专门化的子 Agent 来并行处理不同的子任务。

### 架构设计

Claude Code 的 Sub-Agent 架构遵循了一个简洁的设计原则：

```
主 Agent（用户对话）
  ├── Explore Agent（代码库探索）
  ├── Plan Agent（架构规划）
  ├── General-Purpose Agent（通用任务）
  └── 其他专门化 Agent
```

主 Agent 负责与用户交互、理解意图，并将复杂任务分解为可独立执行的子任务，再委派给最合适的 Sub-Agent 执行。

### Sub-Agent 的类型

#### 1. Explore Agent — 快速代码库探索

Explore Agent 是专为代码搜索优化的轻量级代理。当你需要在大型代码库中查找文件、搜索关键词或理解代码结构时，主 Agent 会将任务委派给 Explore Agent。

**典型使用场景：**
- 通过模式匹配查找文件（如 `src/components/**/*.tsx`）
- 在代码中搜索关键词（如 "API endpoints"）
- 回答关于代码库结构的问题（如 "认证流程是怎么实现的？"）

**特点：**
- 比直接搜索更智能，能根据搜索结果自动调整策略
- 支持设定不同的搜索深度：quick（快速）、medium（中等）、very thorough（深度搜索）
- 只读操作，不会修改任何文件

#### 2. Plan Agent — 软件架构师

Plan Agent 扮演的是一个"软件架构师"角色。在你动手编码之前，它会帮你设计实现方案、识别关键文件、评估技术方案的利弊权衡。

**典型使用场景：**
- 设计新功能的实现方案
- 分析重构策略
- 评估技术方案的优劣

**特点：**
- 返回分步骤的实现计划
- 识别需要修改的关键文件
- 考虑架构层面的权衡取舍
- 只读操作，不直接修改代码

#### 3. General-Purpose Agent — 通用任务执行者

General-Purpose Agent 是最灵活的子代理，它拥有几乎所有工具的访问权限，能够处理需要多步骤、跨文件操作的复杂任务。

**典型使用场景：**
- 搜索并修改分布在多个文件中的代码
- 执行复杂的调查和分析任务
- 运行测试并根据结果进行调整

### 并行执行

Sub-Agent 最强大的特性之一是**并行执行**。主 Agent 可以在一条消息中同时启动多个子代理，各自独立运行：

```
用户: "帮我重构认证模块并添加单元测试"

主 Agent 的决策:
  ├── Agent A: 分析当前认证模块的结构和依赖关系
  ├── Agent B: 调研项目中现有的测试模式和框架
  └── 等待结果后 → 制定重构计划并执行
```

这种并行能力在处理大型任务时能显著提升效率，因为多个独立的调查或操作可以同时进行，而不需要串行等待。

### 后台执行

Sub-Agent 还支持**后台执行模式**。当某个子任务不需要立即获得结果时，主 Agent 可以将其放到后台运行，同时继续与用户交互或处理其他任务：

```
主 Agent: 启动测试 Agent（后台运行）
          ↓ 不等待
主 Agent: 继续编写下一个功能的代码
          ↓
后台 Agent: 测试完成，通知主 Agent
```

### 工作树隔离

对于可能产生代码变更的子任务，Sub-Agent 支持**工作树隔离（Worktree Isolation）**。通过 Git worktree，子 Agent 获得一个独立的仓库副本，在其中自由实验，不会影响主工作目录的状态。如果实验成功，变更会被保留；如果失败，临时工作树会被自动清理。

### 上下文保持与恢复

每个 Sub-Agent 执行完毕后会返回一个 Agent ID。如果后续需要基于之前的结果继续工作，可以通过这个 ID 恢复（resume）子 Agent，它会保留完整的之前的执行上下文，无需从头开始。

## 实际工作流示例

让我们通过一个实际场景来看 Team + Sub-Agent 是如何协作的：

> 假设你正在一个团队项目中，需要为一个 React 应用添加国际化（i18n）支持。

1. **CLAUDE.md 提供上下文**：团队的 CLAUDE.md 中已经记录了项目的技术栈（React + TypeScript）、目录结构约定和代码风格
2. **Plan Agent 设计方案**：主 Agent 先启动 Plan Agent，分析项目结构，设计 i18n 实现方案（选择库、目录结构、翻译文件格式）
3. **Explore Agent 调查现状**：同时启动 Explore Agent，搜索项目中所有硬编码的中文/英文字符串
4. **General-Purpose Agent 执行**：基于规划结果，启动多个 Agent 并行工作 —— 一个安装依赖并配置 i18n 库，一个创建翻译文件，一个逐个组件替换硬编码字符串
5. **测试验证**：最后启动一个后台 Agent 运行测试，确保所有改动没有破坏现有功能

整个过程中，团队的其他成员也可以同时在不同的分支上使用 Claude Code 工作，大家共享相同的 CLAUDE.md 约定，保证代码风格一致性。

## 总结

Claude Code 的 Team 协作和 Sub-Agent 机制代表了 AI 辅助开发的一个重要方向：

- **Team** 解决了"如何让 AI 理解团队约定"的问题，通过 CLAUDE.md 和共享配置，让 AI 成为真正融入团队的成员
- **Sub-Agent** 解决了"如何处理复杂任务"的问题，通过任务分解和并行执行，让 AI 能够高效处理大型软件工程任务

这两个能力的结合，使得 Claude Code 不仅仅是一个代码补全工具，而是一个能够理解上下文、自主规划、并行执行的智能开发伙伴。
